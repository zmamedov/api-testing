# Тестирование REST API

## 1. Введение

В данном репозитории представлен проект тестирования учебного сервиса [JSON Placeholder](https://jsonplaceholder.typicode.com) с использованием Postman.

## 2. Объекты тестирования (Endpoints)

* /posts
* /comments
* /albums
* /photos
* /todos
* /users

## 3. Стратегия тестирования

Тестирование проводится методом "черного ящика" с использованием инструмента Postman.

### Типы проверок:

1. Функциональное тестирование:
    1. Проверка HTTP-методов: `GET`, `POST`, `PUT`, `PATCH`, `DELETE`.

    2. Проверка фильтрации данных через Query Parameters (например, `?userId=1`).

    3. Проверка запросов с вложенными ресурсами (например, `/posts/1/comments`).

2. Тестирование структуры данных (Schema Validation):
    1. Валидация типов полей (ID — integer, Title — string и т.д.).

    2. Наличие обязательных полей в ответе.

3. Негативное тестирование:
    1. Обработка запросов к несуществующим ресурсам.

    2. Отправка некорректных данных в теле запроса.

## 4. Тест-кейсы
